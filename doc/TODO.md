## 阶段一：项目基础架构搭建

**目标**：建立项目目录结构、配置体系和基础工具组件

### 子任务

- [x] 1.1 创建Python虚拟环境
  - 创建虚拟环境：`python -m venv venv`
  - 激活虚拟环境（Windows）：`venv\Scripts\activate`
  - 激活虚拟环境（Linux/Mac）：`source venv/bin/activate`
  - 验证Python版本≥3.10 (Python 3.13.11)

- [x] 1.2 创建项目目录结构
  - 创建 `src/` 目录及其子目录：`infrastructure/`、`domain/`、`services/`、`application/`
  - 创建 `tests/` 目录及其子目录：`unit/`、`integration/`
  - 创建 `config/` 目录存放配置文件模板

- [x] 1.3 创建依赖管理文件并安装依赖
  - 创建 `requirements.txt`，包含：aiohttp、python-dotenv、tomli、matplotlib、numpy、openai、pydantic
  - 安装依赖：`pip install -r requirements.txt`
  - 验证所有依赖安装成功
  - 创建 `.gitignore`，排除 `.env`、日志文件、Python缓存、虚拟环境目录

- [x] 1.4 实现配置加载组件
  - 实现 `.env` 加载器（使用 python-dotenv）
  - 实现 `config.toml` 解析器（支持 Python 3.10+ 使用 tomli）
  - 实现 `prompt.toml` 解析器
  - 实现配置验证逻辑（必填字段检查、类型检查）
  - 定义配置数据模型（Pydantic 或 dataclass）
  - 配置文件位置：`src/infrastructure/config_loader.py`

- [x] 1.5 实现日志组件
  - 实现同时输出到终端和文件的日志处理器
  - 日志文件名格式：`trade_bot_{YYYYMMDD}_{HHMMSS}.log`
  - 日志格式：`[YYYY-MM-DD HH:MM:SS] [级别] [模块名] 内容`
  - 支持日志级别配置（DEBUG/INFO/WARNING/ERROR/FATAL）
  - 文件位置：`src/infrastructure/logger.py`

## 阶段二：基础设施层 - OKX客户端封装

**目标**：自主实现OKX REST和WebSocket客户端，禁止任何第三方OKX SDK

### 子任务

- [x] 2.1 实现OKX签名算法
  - 实现 HMAC-SHA256 签名生成
  - 签名内容格式：`timestamp + method + requestPath + body`
  - 支持实盘和模拟盘切换（模拟盘添加 `x-simulated-trading: 1` 请求头）

- [x] 2.2 实现OKX REST客户端
  - 基于 aiohttp 实现异步HTTP客户端
  - 实现连接池复用，超时10秒
  - 实现指数退避重试（最多3次，间隔1s/2s/4s）
  - 自动处理签名和请求头拼装
  - 实现账户类接口：查询余额、查询仓位、设置持仓模式、设置杠杆、查询交易对信息
  - 实现交易类接口：下单（附带止盈止损）、修改订单、平仓、查询订单详情
  - 实现行情类接口：查询历史K线

- [x] 2.3 实现OKX WebSocket客户端
  - 基于 aiohttp 实现异步WebSocket客户端
  - 支持自动重连（最多5次，间隔1s/2s/4s/8s/16s）
  - 重连失败触发程序退出
  - 重连时自动重新订阅所有频道
  - 订阅K线频道，解析 `confirm` 字段

### 测试标准

- [x] 单元测试：签名算法生成结果与OKX官方示例一致
- [x] 单元测试：REST客户端请求头拼装正确
- [x] 单元测试：WebSocket消息解析正确
- [x] 集成测试：连接OKX模拟盘，验证所有接口调用正常
- [x] 集成测试：验证WebSocket自动重连逻辑

### 验收条件

- [x] 能成功连接OKX模拟盘并调用所有接口
- [x] 签名算法通过官方验证
- [x] WebSocket能稳定接收K线数据
- [x] 网络超时和重试逻辑正常工作

## 阶段三：基础设施层 - LLM客户端与工具组件

**目标**：实现LLM客户端和EMA计算、K线绘图、CSV操作工具

### 子任务

- [x] 3.1 实现LLM客户端
  - 基于 OpenAI 官方异步 Python SDK 实现
  - 超时30秒，支持1次重试
  - 统一处理 API Key、Base URL 配置
  - 支持多模态调用（图片解析）
  - 默认 temperature=0（压缩者可设为0.1）

- [x] 3.2 实现EMA计算工具
  - 基于收盘价计算标准EMA20
  - 平滑系数：`2/(20+1)`
  - 使用 numpy 向量化运算

- [x] 3.3 实现K线绘图工具
  - 生成PNG格式图片，分辨率1080p，确保LLM能清晰解析
  - 包含清晰坐标轴：X轴时间（格式 `YYYY-MM-DD HH:MM`），Y轴价格
  - 蜡烛图上涨红色、下跌绿色，EMA20均线蓝色
  - 无多余装饰元素
  - 生成后保存在内存，转base64后立即释放绘图对象
  - 测试阶段图片保存于本地方便调试

- [x] 3.4 实现CSV存储工具
  - 支持追加写入交易记录
  - 自动创建文件和表头
  - 字段：timestamp、inst_id、position_direction、position_size、entry_avg_price、exit_avg_price、realized_pnl、balance_after_close、order_id

### 测试标准

- [x] 单元测试：EMA计算结果与已知数据一致
- [x] 单元测试：K线绘图生成图片符合规范（分辨率、颜色、坐标轴）
- [x] 单元测试：CSV工具能正确创建文件、写入表头和数据
- [x] 集成测试：LLM客户端能成功发送文本和图片请求

### 验收条件

- [x] EMA20计算准确
- [x] K线图生成正确，base64编码可用
- [x] CSV文件能正确记录交易数据
- [x] LLM客户端能正常调用API

## 阶段七：服务层 - 交易服务

**目标**：实现交易指令执行、仓位管理、风险控制

### 子任务

- [ ] 7.1 实现交易服务核心逻辑
  - 启动时设置所有交易对为单向持仓模式（posMode=net）
  - 设置每个交易对的杠杆
  - 交易操作前重新查询最新仓位和余额

- [ ] 7.2 实现交易指令执行
  - `entry_long`/`entry_short`：市价开仓，附带止盈止损
  - `close_position`：全平并自动取消止盈止损
  - `change_stop`/`change_profit`：修改止盈止损
  - `exit_long`/`exit_short`：仅减仓不平仓
  - 所有请求携带 client_oid 实现幂等

- [ ] 7.3 实现指令参数校验
  - 开仓：止损止盈价格符合配置比例，开仓金额不超过配置
  - 减仓：减仓数量小于当前持仓，禁止全平
  - 修改止盈止损：必须持有对应方向仓位

- [ ] 7.4 实现平仓后处理
  - 触发平仓完成事件
  - 写入交易记录到CSV
  - 清空分析历史
